Este guia fornece um processo passo a passo para migrar um projeto do Next.js App Router para o **TanStack Start**. Respeitamos os recursos poderosos do Next.js e nosso objetivo é tornar essa transição o mais suave possível.

## Passo a Passo (Básico)

Este guia passo a passo fornece uma visão geral de como migrar seu projeto Next.js App Router para o TanStack Start. O objetivo é ajudá-lo a entender os passos básicos envolvidos no processo de migração para que você possa adaptá-los às necessidades específicas do seu projeto.

### Pré-requisitos

Antes de começarmos, este guia assume que a estrutura do seu projeto se parece com isto:

```txt
├── next.config.ts
├── package.json
├── postcss.config.mjs
├── public
│   ├── file.svg
│   ├── globe.svg
│   ├── next.svg
│   ├── vercel.svg
│   └── window.svg
├── README.md
├── src
│   └── app
│       ├── favicon.ico
│       ├── globals.css
│       ├── layout.tsx
│       └── page.tsx
└── tsconfig.json
```

Alternativamente, você pode acompanhar clonando o seguinte [template inicial](https://github.com/nrjdalal/awesome-templates/tree/main/next.js-apps/next.js-start):

```sh
npx gitpick nrjdalal/awesome-templates/tree/main/next.js-apps/next.js-start next.js-start-er
```

Essa estrutura é uma aplicação Next.js básica usando o App Router, que iremos migrar para o TanStack Start.

### 1. Remover o Next.js

Primeiro, desinstale o Next.js e remova os arquivos de configuração relacionados:

```sh
npm uninstall @tailwindcss/postcss next
rm postcss.config.* next.config.*
```

### 2. Instalar as Dependências Necessárias

O TanStack Start utiliza o [Vite](https://vite.dev) e o TanStack Router:

```sh
npm i @tanstack/react-router @tanstack/react-start
```

Para o Tailwind CSS e resolução de imports usando aliases de caminho:

```sh
npm i -D vite @vitejs/plugin-react @tailwindcss/vite tailwindcss vite-tsconfig-paths
```

### 3. Atualizar a Configuração do Projeto

Agora que você instalou as dependências necessárias, atualize os arquivos de configuração do seu projeto para funcionar com o TanStack Start.

- `package.json`

```json
{
  "type": "module",
  "scripts": {
    "dev": "vite dev",
    "build": "vite build",
    "start": "node .output/server/index.mjs"
  }
}
```

- `vite.config.ts`

```ts
// vite.config.ts
import { defineConfig } from "vite";
import { tanstackStart } from "@tanstack/react-start/plugin/vite";
import viteReact from "@vitejs/plugin-react";
import tsconfigPaths from "vite-tsconfig-paths";
import tailwindcss from "@tailwindcss/vite";

export default defineConfig({
  server: {
    port: 3000,
  },
  plugins: [
    tailwindcss(),
    // Permite que o Vite resolva imports usando aliases de caminho.
    tsconfigPaths(),
    tanstackStart({
      srcDirectory: "src", // Este é o padrão
      router: {
        // Especifica o diretório que o TanStack Router usa para suas rotas.
        routesDirectory: "app", // O padrão é "routes", relativo ao srcDirectory
      },
    }),
    viteReact(),
  ],
});
```

Por padrão, `routesDirectory` é definido como `routes`. Para manter a consistência com as convenções do Next.js App Router, você pode defini-lo como `app`.

### 4. Adaptar o Layout Raiz

> O TanStack Start usa uma abordagem de roteamento semelhante ao Remix, com algumas mudanças para suportar estruturas aninhadas e recursos especiais usando tokens. Saiba mais no guia de [Conceitos de Roteamento](/router/latest/docs/framework/react/routing/routing-concepts).

Em vez de `layout.tsx`, crie um arquivo chamado `__root.tsx` no diretório `src/app`. Este arquivo servirá como o layout raiz da sua aplicação.

- `src/app/layout.tsx` para `src/app/__root.tsx`

```tsx
- import type { Metadata } from "next" // [!code --]
import {
  Outlet,
  createRootRoute,
  HeadContent,
  Scripts,
} from "@tanstack/react-router"
import appCss from "./globals.css?url"

- export const metadata: Metadata = { // [!code --]
-   title: "Create Next App", // [!code --]
-   description: "Generated by create next app", // [!code --]
- } // [!code --]
export const Route = createRootRoute({
  head: () => ({
    meta: [
      { charSet: "utf-8" },
      {
        name: "viewport",
        content: "width=device-width, initial-scale=1",
      },
      { title: "TanStack Start Starter" }
    ],
    links: [
      {
        rel: 'stylesheet',
        href: appCss,
      },
    ],
  }),
  component: RootLayout,
})

- export default function RootLayout({ // [!code --]
-   children, // [!code --]
- }: Readonly<{ // [!code --]
-   children: React.ReactNode // [!code --]
- }>) { // [!code --]
-   return ( // [!code --]
-     <html lang="en"> // [!code --]
-       <body>{children}</body> // [!code --]
-     </html> // [!code --]
-   ) // [!code --]
- } // [!code --]
function RootLayout() {
  return (
    <html lang="en">
      <head>
        <HeadContent />
      </head>
      <body>
        <Outlet />
        <Scripts />
      </body>
    </html>
  )
}
```

### 5. Adaptar a Página Inicial

Em vez de `page.tsx`, crie um arquivo `index.tsx` para a rota `/`.

- `src/app/page.tsx` para `src/app/index.tsx`

```tsx
+ import { createFileRoute } from '@tanstack/react-router' // [!code ++]

- export default function Home() { // [!code --]
+ export const Route = createFileRoute('/')({ // [!code ++]
+   component: Home, // [!code ++]
+ }) // [!code ++]

+ function Home() { // [!code ++]
  return (
    <main className="min-h-dvh w-screen flex items-center justify-center flex-col gap-y-4 p-4">
      <img
        className="max-w-sm w-full"
        src="https://raw.githubusercontent.com/TanStack/tanstack.com/main/public/images/logos/splash-dark.png"
        alt="TanStack Logo"
      />
      <h1>
        <span className="line-through">Next.js</span> TanStack Start
      </h1>
      <a
        className="bg-foreground text-background rounded-full px-4 py-1 hover:opacity-90"
        href="https://tanstack.com/start/latest"
        target="_blank"
      >
        Docs
      </a>
    </main>
  )
}
```

### 6. Já migramos?

Antes de executar o servidor de desenvolvimento, você precisa criar um arquivo que definirá o comportamento do TanStack Router dentro do TanStack Start.

- `src/router.tsx`

```tsx
import { createRouter } from "@tanstack/react-router";
import { routeTree } from "./routeTree.gen";

export function getRouter() {
  const router = createRouter({
    routeTree,
    scrollRestoration: true,
  });

  return router;
}
```

> Aqui você pode configurar tudo, desde a funcionalidade padrão de [preloading](/router/latest/docs/framework/react/guide/preloading) até a [obsolescência do cache](/router/latest/docs/framework/react/guide/data-loading).

Não se preocupe se você encontrar alguns erros de TypeScript neste ponto; o próximo passo irá resolvê-los.

### 7. Verificar a Migração

Execute o servidor de desenvolvimento:

```sh
npm run dev
```

Em seguida, acesse `http://localhost:3000`. Você deverá ver a página de boas-vindas do TanStack Start com seu logo e um link para a documentação.

> Se você encontrar problemas, revise os passos acima e certifique-se de que os nomes e caminhos dos arquivos correspondam exatamente. Para uma implementação de referência, veja o [repositório pós-migração](https://github.com/nrjdalal/next-to-start).

## Próximos Passos (Avançado)

Agora que você migrou a estrutura básica da sua aplicação Next.js para o TanStack Start, pode explorar recursos e conceitos mais avançados.

### Conceitos de Roteamento

| Exemplo de Rota                | Next.js                            | TanStack Start            |
| ------------------------------ | ---------------------------------- | ------------------------- |
| Layout Raiz                    | `src/app/layout.tsx`               | `src/app/__root.tsx`      |
| `/` (Página Inicial)           | `src/app/page.tsx`                 | `src/app/index.tsx`       |
| `/posts` (Rota Estática)       | `src/app/posts/page.tsx`           | `src/app/posts.tsx`       |
| `/posts/[slug]` (Dinâmica)     | `src/app/posts/[slug]/page.tsx`    | `src/app/posts/$slug.tsx` |
| `/posts/[...slug]` (Catch-All) | `src/app/posts/[...slug]/page.tsx` | `src/app/posts/$.tsx`     |
| `/api/endpoint` (Rota de API)  | `src/app/api/endpoint/route.ts`    | `src/app/api/endpoint.ts` |

Saiba mais sobre os [Conceitos de Roteamento](/router/latest/docs/framework/react/routing/routing-concepts).

### Rotas Dinâmicas e Catch-All

Recuperar parâmetros de rotas dinâmicas no TanStack Start é simples e direto.

```tsx
- export default async function Page({ // [!code --]
-   params, // [!code --]
- }: { // [!code --]
-   params: Promise<{ slug: string }> // [!code --]
- }) { // [!code --]
+ export const Route = createFileRoute('/app/posts/$slug')({ // [!code ++]
+   component: Page, // [!code ++]
+ }) // [!code ++]

+ function Page() { // [!code ++]
-   const { slug } = await params // [!code --]
+   const { slug } = Route.useParams() // [!code ++]
  return <div>My Post: {slug}</div>
}
```

> Nota: Se você criou uma rota catch-all (como `src/app/posts/$.tsx`), pode acessar os parâmetros via `const { _splat } = Route.useParams()`.

Da mesma forma, você pode acessar `searchParams` usando `const { page, filter, sort } = Route.useSearch()`.

Saiba mais sobre as [Rotas Dinâmicas e Catch-All](/router/latest/docs/framework/react/routing/routing-concepts#dynamic-route-segments).

### Imagens

O Next.js usa o componente `next/image` para imagens otimizadas. No TanStack Start, você pode usar o pacote chamado [Unpic](https://unpic.pics/) para funcionalidade semelhante
e uma substituição quase direta.

```tsx
import Image from "next/image"; // [!code --]
import { Image } from "@unpic/react"; // [!code ++]
function Component() {
  return (
    <Image
      src="/path/to/image.jpg"
      alt="Description"
      width="600" // [!code --]
      height="400" // [!code --]
      width={600} // [!code ++]
      height={400} // [!code ++]
    />
  );
}
```

### Fontes

```tsx
- import { Inter } from "next/font/google" // [!code --]

- const inter = Inter({ // [!code --]
-   subsets: ["latin"], // [!code --]
-   display: "swap", // [!code --]
- }) // [!code --]

- export default function Page() { // [!code --]
-   return <p className={inter.className}>Font Sans</p> // [!code --]
- } // [!code --]
```

Em vez de `next/font`, use a abordagem CSS-first do Tailwind CSS. Instale fontes (por exemplo, do [Fontsource](https://github.com/fontsource/fontsource)):

```sh
npm i -D @fontsource-variable/dm-sans @fontsource-variable/jetbrains-mono
```

Adicione o seguinte ao `src/app/globals.css`:

```css
@import "tailwindcss" source("../");

@import "@fontsource-variable/dm-sans"; /* [!code ++] */
@import "@fontsource-variable/jetbrains-mono"; /* [!code ++] */

@theme inline {
  --font-sans: "DM Sans Variable", sans-serif; /* [!code ++] */
  --font-mono: "JetBrains Mono Variable", monospace; /* [!code ++] */
  /* ... */
}

/* ... */
```

```tsx

+ function Page() { // [!code ++]
-   const data = await fetch('https://api.vercel.app/blog') // [!code --]
-   const posts = await data.json() // [!code --]
+   const posts = Route.useLoaderData() // [!code ++]
```
