# Diretorio `bin/`

## Visao Geral

O diretorio `bin/` contem o ponto de entrada da aplicacao. Possui um unico arquivo responsavel por inicializar a conexao com o banco de dados e iniciar o servidor HTTP.

---

## Estrutura

```
bin/
└── server.ts    # Ponto de entrada da aplicacao
```

---

## `bin/server.ts`

Este arquivo e o primeiro a ser executado quando a aplicacao inicia. Ele realiza duas operacoes em sequencia:

1. Conecta ao MongoDB via `MongooseConnect()`
2. Inicializa o servidor Fastify via `kernel.ready()` e `kernel.listen()`

### Codigo fonte

```typescript
import { MongooseConnect } from '@config/database.config';
import { Env } from '@start/env';
import { kernel } from '@start/kernel';

async function start(): Promise<void> {
  try {
    await kernel.ready();

    await kernel.listen({ port: Env.PORT, host: '0.0.0.0' });
    console.info(`HTTP Server running on http://localhost:${Env.PORT}`);
  } catch (err) {
    console.error('Error starting server:', err);
    process.exit(1);
  }
}

MongooseConnect().then(() => {
  console.info('Mongoose connected');
  console.info('url: ', Env.DATABASE_URL);
  start();
});
```

### Fluxo de execucao

```
1. MongooseConnect()
   ├── Conecta ao MongoDB usando DATABASE_URL e DB_NAME
   ├── Registra todos os modelos Mongoose
   └── Em caso de erro: desconecta e encerra com process.exit(1)

2. start()
   ├── kernel.ready()    → Inicializa todos os plugins Fastify
   ├── kernel.listen()   → Inicia o servidor HTTP
   │   ├── port: Env.PORT (padrao: 3000)
   │   └── host: '0.0.0.0' (aceita conexoes de qualquer interface)
   └── Em caso de erro: encerra com process.exit(1)
```

### Detalhes

- **`kernel.ready()`**: Garante que todos os plugins registrados no Fastify estejam carregados antes de aceitar requisicoes. Isso inclui CORS, JWT, Swagger, DI, e o bootstrap dos controllers.
- **`kernel.listen()`**: Inicia o servidor HTTP na porta definida pela variavel de ambiente `PORT` e no host `0.0.0.0`, o que permite que o servidor aceite conexoes de qualquer interface de rede (necessario para uso em containers Docker).
- **Tratamento de erros**: Tanto a conexao com o banco quanto a inicializacao do servidor possuem blocos try/catch. Em caso de falha, a aplicacao e encerrada com codigo de saida 1.

## Scripts npm

O arquivo `bin/server.ts` e referenciado por tres scripts no `package.json`:

### Desenvolvimento

```bash
npm run dev
```

Executa:
```bash
node --import @swc-node/register/esm-register --watch bin/server.ts
```

- `--import @swc-node/register/esm-register`: Registra o SWC como compilador TypeScript em tempo real, com suporte a ESM
- `--watch`: Reinicia automaticamente o servidor ao detectar mudancas nos arquivos

### Producao

```bash
npm start
```

Executa:
```bash
node build/bin/server.js
```

- Executa o codigo ja compilado pelo tsup no diretorio `build/`

### Build

```bash
npm run build
```

Executa:
```bash
tsc -b && tsup
```

- `tsc -b`: Verifica tipos TypeScript (type-checking apenas, sem emitir)
- `tsup`: Empacota o codigo para producao em formato ESM, target ES2024, saida em `build/`

## Dependencias

O `bin/server.ts` depende de:

| Import | Origem | Funcao |
|---|---|---|
| `MongooseConnect` | `@config/database.config` | Funcao para conectar ao MongoDB |
| `Env` | `@start/env` | Variaveis de ambiente validadas |
| `kernel` | `@start/kernel` | Instancia do Fastify configurada |

---

## Variaveis de ambiente utilizadas

| Variavel | Descricao | Padrao |
|---|---|---|
| `PORT` | Porta do servidor HTTP | `3000` |
| `DATABASE_URL` | URL de conexao com MongoDB | _(obrigatoria)_ |
| `DB_NAME` | Nome do banco de dados | `lowcodejs` |
